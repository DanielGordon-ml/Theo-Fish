Testing Strategy

Layered testing with integration gates, real Neo4j, and mocked LLM boundaries

Testing Principles
Test Before Code

Per CLAUDE.md: test file written before implementation. Every module gets its test file first.

Real Neo4j for Integration

Integration tests run against a real Neo4j instance (Docker testcontainer). No mocking the graph database — it's the source of truth.

Mocked LLM at Unit Boundary

Unit tests mock the LLM client. Integration tests may use recorded fixtures (VCR-style) for reproducible LLM responses.

Schema as Test Fixture

Tests load the real schema.yaml — not a test copy. If schema changes break tests, that's a signal, not a nuisance.

LAYER 2 GATE — Integration Test Milestone
Before building Layer 3, this test must pass:

test_layer2_gate.py

1. Start Neo4j testcontainer with schema constraints + vector index
2. Write a handcrafted :Concept ("markov-decision-process", type: structure) with embedding
3. Write a handcrafted :Claim ("theorem-1-1", type: theorem) with ABOUT → concept
4. Write a :Source and :Provenance with FORMULATION_OF + SOURCED_FROM edges
5. Write all edge types (GENERALIZES, DEPENDS_ON, ABOUT, etc.) with attributes
6. Read everything back, confirm schema constraints hold
7. Confirm field ownership: MERGE same concept with different human-owned field → original preserved
8. Confirm vector similarity search returns the concept
9. Confirm node-level transaction isolation: failed claim doesn't affect written concept

This validates that models/ and graph/ agree on the Neo4j schema before extraction produces real data.

END-TO-END SMOKE TEST
test_e2e_smoke.py — runs against real Neo4j, mocked LLM with recorded fixtures

1. Load a real cleaned paper JSON (fixture from Data/cleaned/)
2. Run full pipeline: split → P1 → merge → P2 → P3a → P3b → validate → write
3. Query Neo4j: confirm :Concept, :Claim, :Source, :Provenance nodes exist
4. Confirm all edge types present with correct attributes
5. Run export → confirm vault files in correct folders
6. Edit a human-owned field in vault → run sync-back → confirm Neo4j updated
7. Re-export → confirm human edit preserved

This is the acceptance test for the entire system.
Test Infrastructure
Neo4j Testcontainer

Docker-based Neo4j for integration tests. Shared fixture creates constraints + indexes on startup. Each test gets a clean database (MATCH (n) DETACH DELETE n between tests). Same neo4j:5.26-community image as production.

LLM Fixtures

Recorded LLM responses (VCR-style JSON files) for reproducible extraction tests. Unit tests use simple mocks. Integration tests use recorded fixtures from a real DeepSeek run. Fixture files live in tests/fixtures/llm/.

Test Markers

@pytest.mark.unit — no external deps
@pytest.mark.integration — requires Neo4j
@pytest.mark.e2e — full pipeline smoke
@pytest.mark.slow — tests taking >10s
CI runs unit always, integration on PR, e2e on merge to main.

Graph Fixtures

Handcrafted Neo4j fixtures in tests/fixtures/graph/ — pre-built concept/claim/source nodes and edges for tests that need existing vault state (dedup tests, export tests, sync-back tests).
