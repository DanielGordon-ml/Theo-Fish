Module Responsibilities — Single Responsibility Boundaries
Module Responsibility Depends On Depended On By
config/ Load env vars, parse schema.yaml into typed objects — Everything
models/ Pure data structures — no behavior, no I/O — Everything
llm/ DeepSeek API calls, retry logic, token tracking, JSON parsing config extraction
extraction/ All LLM-powered extraction: concepts, claims, edges. Prompt generation from schema. config, models, llm orchestrator
dedup/ Embedding computation, fuzzy matching, dedup cascade orchestration config, models, llm, graph orchestrator
graph/ Neo4j read/write, connection management, schema validation config, models dedup, vault, orchestrator
vault/ Obsidian export (Neo4j → md), sync-back (md → Neo4j), frontmatter/body generation config, models, graph orchestrator
orchestrator/ Pipeline coordination, batch processing, CLI entry point All modules __init__, __main__
Key Design Constraints (from CLAUDE.md)
File Size

No file exceeds 600 lines. Split at 500. Each file in extraction/ handles one pass. Edge enricher and cross-section linker are separate files despite both being "Pass 3."

Functions

Max 20 lines per function, max 3 params. Orchestrator stages are separate functions, not one monolith. Options objects for anything complex.

Naming

Files: kebab-case. Classes: PascalCase. Functions: camelCase. Constants: UPPER_SNAKE_CASE. Booleans: is/has/can/should prefix.

Testing

Test file written BEFORE implementation. Mirror source paths. Cover happy path, edge cases, error cases. Run single test file, not full suite.

Documentation

Every file has a header comment. Every export has JSDoc. Comments explain WHY not WHAT.

Error Handling

Never swallow errors. Always include context. GraphBuilderError, ValidationError, DedupConflictError with cause chaining.
